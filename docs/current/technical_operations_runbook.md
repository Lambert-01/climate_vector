# DengueEW-GL Technical Operations Runbook

## Implemented controls

- JWT login and role authorization for `admin`, `technical_reviewer`, `field_officer`, `lab_analyst`, `data_manager`, and `viewer`.
- Production write endpoints fail closed when neither a valid JWT nor integration key is supplied.
- Neon-backed dataset registry and audit log.
- Managed JPEG, PNG, and WebP community evidence stored in Neon with size limits and SHA-256 checksums.
- Alert-linked response actions with controlled status transitions and mandatory completion evidence.
- Genomic sample and artifact registries plus a real, input-gated neighbor-joining pipeline.
- Request IDs, structured server error logging, optional Sentry, operational status, scheduled climate ingestion, uptime checks, and verified database dumps.

## Local activation

Apply the schema and seed source provenance:

```bash
source .venv/bin/activate
alembic upgrade head
python scripts/pipelines/16_seed_operational_registry.py
```

Create or rotate the first administrator only after reviewing `.env`:

```bash
python scripts/pipelines/17_bootstrap_admin.py
```

The bootstrap script requires `BOOTSTRAP_ADMIN_EMAIL`, `BOOTSTRAP_ADMIN_PASSWORD`, and `BOOTSTRAP_ADMIN_NAME`. Remove `BOOTSTRAP_ADMIN_PASSWORD` from Render after the account has been created. Additional accounts should be created through `POST /api/auth/users` by an authenticated administrator.

Refresh current climate rows:

```bash
python scripts/pipelines/18_refresh_open_meteo_climate.py
```

Run the platform:

```bash
bash scripts/stop_all.sh
bash scripts/run_all.sh
```

Open `http://127.0.0.1:5173` and use the account button in the top bar.

## Render

Set these environment variables in the `rcvis-api` service:

- `DATABASE_URL`
- `DATABASE_SYNC_URL`
- `API_CORS_ORIGINS=https://climate-vector-web.vercel.app`
- `JWT_SECRET` generated by Render
- `OPERATOR_API_KEY` as a long random value
- `BOOTSTRAP_ADMIN_EMAIL`, `BOOTSTRAP_ADMIN_NAME`, and temporarily `BOOTSTRAP_ADMIN_PASSWORD`
- `SENTRY_DSN` when a Sentry project is available
- `PROJECT_ENV=production`

The start command applies pending Alembic migrations before Uvicorn starts. Open a Render Shell once and run the registry and administrator scripts. Then delete `BOOTSTRAP_ADMIN_PASSWORD` and redeploy. The free Render plan sleeps and can take over 30 seconds to wake. Select a paid always-on instance for an operational pilot; an uptime check is monitoring, not a substitute for an always-on service.

## Vercel

Set `VITE_API_BASE=https://rcvis-api.onrender.com/api` in the production project and redeploy. No database or JWT secret belongs in Vercel because the browser communicates only with the API.

## GitHub Actions

Create repository secrets `DATABASE_URL` and `DATABASE_SYNC_URL`. Create repository variable `API_HEALTH_URL=https://rcvis-api.onrender.com/api/health`. Enable Actions for:

- `API uptime check`: hourly API and database verification.
- `Verified Neon backup`: daily custom-format dump plus `pg_restore --list` verification and 14-day artifact retention.
- `Scheduled climate ingestion`: daily Open-Meteo district refresh.

Periodically perform a full restore into an isolated Neon branch. Catalog verification proves that a dump is readable, while a test restore proves recovery.

## Genomics

Do not run lineage or phylogeny analysis without laboratory-reviewed sequences. For an aligned consensus FASTA:

```bash
python scripts/pipelines/19_build_dengue_phylogeny.py reviewed_alignment.fasta
```

Review the generated Newick tree and manifest, then register the artifact through `/api/dengue/genomic-artifacts`. No sequence, lineage, serotype, or tree is simulated when inputs are absent.

## Remaining external choices

- Create the first administrator account after credential review.
- Add a Sentry DSN or another approved error-tracking service.
- Upgrade Render for always-on availability before field operations.
- Approve backup retention, data protection, image consent, user roles, and genomic review procedures with the PI and host institution.
